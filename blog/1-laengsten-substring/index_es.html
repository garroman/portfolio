Encontrar el tamaño del substring más largo

16-02-2023

Este post tratará brevemente sobre un problema que encontrè en Leetcode.

El problema nos pide crear una función que encuentre el tamaño del substring más largo sin carácteres repetidos en un string s dado.

Ejemplos:

Input: s = “abcd”
Output: 4
Explicación: La respuesta es todo el string “abcd”, el cúal tiene un largo de 4.

Input: s = “bbbbb”
Output: 1
Explicaciòn: La respuesta es el substring “b”, el cúal tiene un largo de 1.

Input: s = “pwwkew”
Output: 3
Explicación: La respuesta es el substring “wke”, que tiene un largo de 3.

Soluciones:

Para resolver este problema es necesario encontrar todos los substrings sin carácteres repetidos.

Explicaré a continuación dos formas de resolverlo.

Todos los ejemplos de código están en Java y representan la solución más no bien un programa listo para ser ejecutado.

Primera solución:

Para este método se buscará cada substring sin carácteres repetidos del string

Para ello es necesaria una variable que contenga el máximo tamaño encontrado y será llamada maxLength.

Es necesario un primer bucle que permita recorrer cada carácter del string.

Y un segundo bucle contenido en el primero.

Se denotará al carácter actual como innerChar y el final del string como right.

El segundo bucle será terminado cuando se encuentre un carácter que ya esté en el substring actual o cuando se haya llegado al final de la cadena.

Luego se actualiza el valor maxLength en caso de que este sea menor al valor del último substring encontrado.

De esta forma se puede llegar a encontrar este valor el cúal representa el máximo largo del substring sin carácteres repetidos en un string.

Código:

int maxLength = 0;
int right = 0;

for (int i = 0; i < s.length(); i++) {
	for (right = i + 1; right < s.length(); right++) { 
		char innerChar = s.charAt(right);
		if (s.indexOf(innerChar) != right) {
			break;
		}
	}
	maxLength = Math.max(maxLength, right - i);
}
return maxLength;

Conclusión de la primera solución:

Esta solución es suficiente para resolver el problema pero frecuentemente no lo es para otros casos de Input. Es por eso que encontrar la solución es solo el primer paso.

Se buscan soluciones que sirvan para el contexto en el que se desarrolla el problema y en este caso esta solución es demasiado lenta.

Es una solución lenta ya que se necesitan de dos bucles para poder iterar mas de una vez cada caracter en el string. Para mejorar la velocidad del álgoritmo sería optimo recorrer el string solo una vez.

Esta última propuesta será implementada en la segunda solución.

Segunda solución:

El método empleado en esta solución se lo conoce en ingles como “Sliding Window Technique”.

Se selecciona una parte del string como la parte más larga sin carácteres repetidos y se la actualiza al momento de encontrar un carácter repetido.

Luego de cada carácter se actualizará la variable maxLength en caso de que el substring actualizado sea más largo y el carácter no este repetido en este.

Para eso utilizaremos un HashMap, asignandolo a la variable que se llamará charMap. Este mapa nos ayudará a mantener un registro de cada carácter que ya haya sido “visto”.

Para la implementación de la solución son necesarios también dos punteros que serán asignados a variables llamadas left y right respectivamente.

Código:

int maxLength = 0;  
Map<Character, Integer> charIndexMap = new HashMap<>();  
int left = 0;  

for (int right = 0; right < s.length(); right++) {
	Character actualChar = s.charAt(right);
	int index = charIndexMap.getOrDefault(actualChar, -1);
	// ya contenido en el substring.
	if (index >= left) {
		left = index + 1;
	} else {
		maxLength = Math.max(maxLength, right - left + 1);
	}
	charIndexMap.put(actualChar, right);
}

return maxLength;
